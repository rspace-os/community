# Code Contributions

RSpace runs on a Java/Spring backend with a React frontend (with some parts still migrating from jQuery). Contributions are welcome across backend, frontend, API, and infrastructure.

## Before You Start

- Read [Getting Started](getting-started.md) if you haven't already
- Reach out before starting significant work — opensource@researchspace.com or [office hours](../community.md)
- You'll need to sign a **CLA** when you open your first pull request. See [Licenses & CLA](../policies/licenses.md)

## Set Up Your Environment

Everything you need to get RSpace running locally is in the developer documentation:

→ **[Developer Getting Started Guide](https://github.com/rspace-os/rspace-web/tree/main/DevDocs/DeveloperNotes/GettingStarted)**

This covers dependencies, database setup, building the project, running RSpace locally, and IDE configuration.

## Coding Standards

Follow the project's coding standards for Java, frontend, and API work:

→ **[Coding Standards](https://github.com/rspace-os/rspace-web/blob/main/DevDocs/DeveloperNotes/GettingStarted/CodingStandards.md)**

Standards cover naming conventions, code style, testing requirements, and security practices.

## Contribution Workflow

1. **Discuss** — talk with maintainers before starting (see above)
2. **Fork** — fork [rspace-web](https://github.com/rspace-os/rspace-web) or relevant repository you're working on and create a descriptive branch (`fix/`, `feature/`, `docs/`)
3. **Develop** — make focused commits, follow coding standards, keep your branch updated with main
4. **Test** — run existing tests, add tests for new functionality; see [testing requirements](https://github.com/rspace-os/rspace-web/blob/main/DevDocs/DeveloperNotes/GettingStarted/CodingStandards.md#testing)
5. **Submit a PR** — clear description of what changed and why, link related issues, include screenshots for UI changes
6. **Review** — maintainers review within their capacity; address feedback, be patient with complex changes
7. **Merge** — once approved, maintainers merge and your contribution goes into the next release 🎉

## Getting Help

If you're stuck, the developer docs are the first place to look:

→ **[DevDocs](https://github.com/rspace-os/rspace-web/tree/main/DevDocs)**

Beyond that: comment on your PR, ask at [office hours](../community.md), or email opensource@researchspace.com.
