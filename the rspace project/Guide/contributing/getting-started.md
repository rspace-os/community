# Getting Started as a Contributor

Welcome to the RSpace community! This page covers what every contributor needs to know before diving in — whatever form your contribution takes.

---

## 1. Read the Code of Conduct

Our **[Code of Conduct](../policies/code-of-conduct.md)** applies to all contribution types and all community spaces. Please read it first.

---

## 2. Reach Out Before You Invest Significant Time

For anything beyond filing an issue, it's worth making contact before you invest significant effort. This avoids duplicate work, ensures your idea aligns with the roadmap, and gets you any guidance you need upfront.

- 📧 opensource@researchspace.com
- 🗓 [Office hours](../community.md) — the easiest way to talk things through in real time

---

## 3. Understand the CLA

If your contribution involves committing code or content to an RSpace repository, you'll need to sign a **Contributor License Agreement (CLA)**. This is not required for filing issues or leaving comments.

The CLA process is handled automatically when you open your first pull request. See [Licenses & CLA](../policies/licenses.md) for details.

---

## Choose Your Path

Not sure where to start? Pick the option that fits you best:

| I want to… | Start here |
|---|---|
| Report a bug or suggest a feature | [Open an issue](https://github.com/rspace-os/rspace-web/issues) on rspace-web |
| Contribute code | [Code contributions](code.md) |
| Give feedback on the UI or help with UX | [UX & Design](ux-design.md) |
| Build an integration with another tool | [Integrations](integrations.md) |
| Improve documentation | [Documentation](documentation.md) |
| Share a lab template | [Templates](templates.md) |
| Help translate RSpace | [Translation](translation.md) |

## Recognition

Contributors are recognised through:
- Git commit history and GitHub profiles
- Release notes acknowledgments
- Community shoutouts in office hours and newsletters
- Contributor listings (coming soon)

---
## Getting Help

Stuck? Have questions?

- Check the specific contribution guide for your area
- Ask in [office hours](../community.md)
- Email opensource@researchspace.com
- Search GitHub issues for similar questions

We're here to help you find your footing.
