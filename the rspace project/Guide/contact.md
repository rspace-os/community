# Connecting with the RSpace Open Source Project

## Communication Tips

- **Be specific** — provide context and details; use existing issue templates where available
- **Check first** — search existing issues and documentation before reaching out
- **Be patient** — we're a small team balancing multiple priorities
- **Be respectful** — follow our [Code of Conduct](policies/code-of-conduct.md)

## What Would You Like to Do?

- **Report a bug or suggest a feature** → [Open an issue](https://github.com/rspace-os/rspace-web/issues) on rspace-web
- **Found a security vulnerability** → Follow our [security disclosure process](policies/security.md)
- **Contribute code, docs, templates, or a design** → Start with [Getting Started](contributing/getting-started.md)
- **Attend a community event or office hours** → See the [calendar](calendar.md)
- **Give general feedback about RSpace to the core team** → [Feedback form](https://forms.office.com/e/fBTBtuPsyb)
- **Connect in person** → [Community events](https://www.researchspace.com/community#events) on researchspace.com
- **Deployment support or enterprise services** → [researchspace.com/contact](https://researchspace.com/contact) or our [solutions page](https://researchspace.com/pricing)
- **Anything else** → opensource@researchspace.com
